API

You can interface with Yucando programmatically through our REST API. The documentation is shown below. You will need to authenticate each request using either a username:password combination, or you will need to attach a TOKEN *jwt* to your request.

Your jwt is:

Endpoints

GET /task/id/

Return task with specified Mongo ID

DELETE /task/

Delete all tasks with Mongo ID

GET /task

Return all tasks as JSON array

POST /task

Create a new task, return created record in JSON array. The JSON payload may consist of the following:

  • taskname (String) The name of your task.
  • timeEstimate (Number Number of seconds estimated to complete the task.
  • points (Number) Points awarded at completion of task.
  • tags (Array of Strings) Tags for searching.
  • notes (String) Notes associated with the task.
  • due (Date) Date when the task is due.
  • defer (Date) Task is invisible until defer date has passed.
  • timeCreated (Date) Time at which the task was created. Defaults to current timestamp.
  • timeUpdated (Date) Time at which the task was last updated. Defaults to current timestamp.
  • timeCompleted (Date) Time at which the task was completed.
  • repeat (Mixed) Repeat schema for task.
  • project (String) Project to which this task belongs
  • visibility (String) Visibility of task. Can be from a list of public (all can see), private (only you can see), members (only project members can see) and <group>, a group which you specify that has certain privileges.
  • onComplete (Mixed) A trigger that is executed when you complete a task, such as notifying a friend by email.
  • punches (Array) Time punches logged for the task.
